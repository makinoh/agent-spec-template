# prompts/evaluations/ — プロンプト回帰テスト（evals）

プロンプト資産の品質を守るための**評価（eval）**を格納します。
プロンプトはレビュー対象資産であり、劣化を検知できる状態を保ちます（2026 トレンド：プロンプトを「テストする」）。

## 各 eval の記述

| 項目 | 内容 |
| --- | --- |
| 対象プロンプト | `id`（[prompts/](../README.md) 資産） |
| 入力 | 合成データのみ（本番 PII 禁止＝MUST NOT） |
| 期待出力／判定 | 合否基準（完全一致・含意・スキーマ適合 等） |
| 既知の失敗 | 回帰の起点となる過去の失敗例 |

## 運用

- 重要なプロンプト変更は対応する eval の更新を伴うべきです（SHOULD）。
- 失敗時はゲートを緩めず原因（プロンプト）側を修正します（MUST NOT 回避。憲章「自己修正ループの防止」）。
- 自動実行の配線（CI でのプロンプト eval）は採用組織が各スタックで実装します（[scripts/checks/prompts.sh](../../scripts/checks/prompts.sh) を起点に拡張）。
