# セキュリティポリシー

## 脆弱性の報告

セキュリティ上の問題を見つけた場合は、**公開 Issue を作成せず**、GitHub の
「**Security**」タブ →「**Report a vulnerability**」（Private Vulnerability Reporting）から
非公開で報告してください。

- 影響範囲・再現手順・想定されるリスクを可能な範囲で記載してください。
- 受領後、可能な限り速やかに確認し、対応方針を返信します。

## 方針

- 本リポジトリは秘密情報を含めません。秘密情報のハードコードは CI（シークレットスキャン）で検出・ブロックします。
- 依存の既知脆弱性は CI（依存スキャン。CVSS 7.0 以上で失敗）でブロックします。
- 詳細な基準は [standards/security-standards.md](standards/security-standards.md) を参照してください。

## 対象バージョン

本テンプレートはドラフト（`v0.1.0`）です。最新の `main` ブランチを対象とします。
