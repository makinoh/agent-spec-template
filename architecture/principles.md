# アーキテクチャ原則（Architecture Principles）

* Version: 0.1.0（Proposed / ドラフト）
* Date: 2026-04-01
* 上位規範: constitution.md（開発憲章「アーキテクチャの完全性」）

本書は、本プロジェクトのアーキテクチャ原則の正本（SSoT）です。各原則は設計レビューと Class B 判定の根拠になります。

---

## 原則（初期セット・採用組織が拡張）

1. **疎結合・高凝集**: モジュールは単一の責務を持ち、境界を越える結合を最小化する（SHOULD）。
2. **明示的な依存方向**: 依存は一方向とし、循環依存を導入しない（MUST NOT。architecture-standards.md「1.」）。
3. **契約による境界通信**: 境界越えは公開された契約（API・イベント）を通じて行う（SHOULD）。
4. **置換可能性**: 外部依存はアダプタで隔離し、差し替え可能にする（SHOULD）。
5. **可逆性・観測性を既定に**: 本番影響変更はロールバック手段と観測性を備える（SHOULD。憲章）。
