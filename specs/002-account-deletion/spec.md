# 仕様（Specification）: アカウントの削除（忘れられる権利）

> サンプル機能（テンプレートの実演。採用時は実要求に置換）。
> 本書は「何を・なぜ（What/Why）」の正本。How は plan.md、判断根拠は ADR-0002。
> ドメイン知識は knowledge/domain/account-data.md を参照。

## 1. 概要 / 問題

- 背景・課題: 利用者が自分のアカウントを自身で削除する手段がなく、消去権（GDPR 第17条）に応えられない。
- ゴール: 認証済み利用者が、自分自身のアカウントを削除でき、一定の猶予期間内は取り消せる。
- 非ゴール: 管理者による強制削除／他ユーザの削除／バックアップ媒体からの即時消去（運用で別途）。

## 2. ユーザシナリオ

- **US-1**: 登録利用者として、自分のアカウント削除を要求できる。
  - Given ログイン済み / When 削除を要求 / Then アカウントが削除予定（ソフト削除）になる。
- **US-2**: 利用者として、猶予期間内であれば削除を取り消せる。
  - Given 削除予定 / When 猶予期間内に取消 / Then アカウントが復元される。
- **US-3**: 利用者として、他人のアカウントは削除できない。
  - Given 利用者 A / When 任意の操作 / Then 削除できるのは A 自身のみ。

## 3. 機能要求（Functional Requirements）

- **FR-1**: システムは、認証済み利用者の要求により、当該利用者のアカウントをソフト削除しなければならない（MUST）。
- **FR-2**: システムは、要求者自身のアカウントのみを削除し、他ユーザを削除してはならない（MUST NOT）。
- **FR-3**: システムは、猶予期間（既定 30 日）内の取消を許容しなければならない（MUST）。
- **FR-4**: システムは、猶予期間経過後にハード削除（不可逆）を実行しなければならない（MUST）。
- **FR-5**: システムは、削除・取消・ハード削除の各事象を監査ログに記録するべきである（SHOULD）。
- **FR-6**: システムは、法的保持義務のある項目を、猶予期間後も定義された期間保持してよい（MAY）。

## 4. 非機能要求・制約（NFR / Constraints）

- **データ機密区分**: 対象は個人データ（PII）= **Restricted**（standards/security-standards.md「1.」）。本番 PII を AI／外部 AI に入力してはならない（MUST NOT）。
- **可逆性**: ハード削除は不可逆操作（憲章「変更の可逆性」）。ハード削除の実行は **Class A**。猶予期間（ソフト削除）で可逆性を確保する。
- **認可**: 自己データのみ。認可判定は機能の中核（**Class A** 相当）。
- **整合性**: 削除予定中のエクスポート（feature 001）の可否を定義する。
