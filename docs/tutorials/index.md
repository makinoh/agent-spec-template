# チュートリアル（実践ハンズオン）

このシリーズは、**1 つの小さな題材を最初から最後まで**通すことで、テンプレートの開発サイクルを体得します。
[コンセプト](../concepts/index.md) が「なぜ」なら、こちらは「**どうやって**」です。

## 通して作る題材

> **題材:** 「**メモにタグを付けて検索する**」機能（`search-notes-by-tag`）。
> ログイン済みユーザーが、自分のメモを 1 つ以上のタグで絞り込んで一覧できる、という小さな機能です。

この 1 題材を、6 つのチュートリアルで段階的に作り込みます。

```mermaid
flowchart LR
    T1["1. プロジェクト作成"] --> T2["2. ADR"]
    T2 --> T3["3. 仕様（Spec）"]
    T3 --> T4["4. 実装"]
    T4 --> T5["5. レビュー"]
    T5 --> T6["6. 運用"]
```

## 各チュートリアル

| # | タイトル | できるようになること | 目安 |
| --- | --- | --- | --- |
| 1 | [テンプレートからプロジェクト作成](01-create-project.md) | テンプレ複製・プロファイル選択・初期結線 | 20分 |
| 2 | [ADRを作成する](02-write-adr.md) | 設計判断を ADR に残し、索引・ゲートを通す | 20分 |
| 3 | [仕様（Spec）を作成する](03-write-spec.md) | What/Why を spec に書き、曖昧さを解消する | 20分 |
| 4 | [Claude Codeで実装する](04-implement.md) | plan→tasks→実装、テストとゲート通過 | 30分 |
| 5 | [レビューする](05-review.md) | クラス判定・ラベル・承認・DoD 確認 | 25分 |
| 6 | [運用する](06-operate.md) | リリース・障害対応・憲章改正・計測 | 25分 |

## 前提

- [前提環境を整える](../getting-started/prerequisites.md) を済ませている
- [30分クイックスタート](../getting-started/quickstart.md) を一度やってみた（推奨）

> **進め方:** 上から順にどうぞ。各回の最後に「確認」チェックがあります。
> 詰まったら [トラブルシューティング](../troubleshooting.md) と [FAQ](../faq.md) へ。
