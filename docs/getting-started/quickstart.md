# 30分クイックスタート

> **ゴール:** 30 分で **テンプレートを動かし、最初の成果物（自分の `spec.md`）を 1 本** つくる。
> **前提:** [前提環境を整える](prerequisites.md) を済ませていること（最低限 Git、できれば AI エージェント）。

> **⚠️ 最初に 1 つだけ約束:** 本番の個人データ・顧客機密・APIキーなどの秘密情報を、AI に貼り付けないでください。
> これはどのプロファイルでも緩められない**絶対ルール**です（理由は [AIDD](../concepts/ai-driven-development.md) で解説）。

このページは時間割形式です。上から順にどうぞ。

---

## 0:00–0:05 ｜ テンプレートを取得する

**GitHub で複製する場合（推奨）:** リポジトリの **「Use this template」→ Create a new repository** で自分のリポジトリを作り、クローンします。

```bash
git clone https://github.com/<your-account>/<your-repo>.git
cd <your-repo>
```

> 単に中身を見たいだけなら、テンプレート本体をそのままクローンしても構いません。

---

## 0:05–0:10 ｜ 「手すり」を確認する

このテンプレートには、書き方の手本になる**サンプル**が同梱されています。まず眺めて感覚をつかみます。

| 見るもの | これは何か |
| --- | --- |
| `README.md` | 全体像 |
| `AGENTS.md` | AI エージェントの入口（ルールの要約） |
| `specs/001-user-profile-export/spec.md` | 「何を・なぜ」を書いた **spec の実例** |
| `adr/adr-0000-adr-format-and-governance.md` | 「なぜこの設計か」を書いた **ADR の実例** |

> 詳しい読み解きは [実例で学ぶ](../examples/index.md) にあります。いまは「こういう粒度で書くのか」と眺めるだけで OK。

---

## 0:10–0:25 ｜ 最初の成果を出す

ここが本番です。手元の環境に応じて **トラック A / B のどちらか** を選びます。

### トラック A ｜ AI エージェントがある（おすすめ）

Claude Code などの **spec-kit 対応エージェント** をこのリポジトリで開き、最初の仕様を **AI に下書きさせます**。

1. エージェントをリポジトリのルートで起動する。
2. 次のように `/speckit.specify` を実行する（例として小さな機能を 1 つ）。

   ```text
   /speckit.specify タグでメモを検索する機能。
   ログイン済みユーザーが、自分のメモを 1 つ以上のタグで絞り込んで一覧できる。
   他人のメモは検索結果に出ないこと。
   ```

3. エージェントが `specs/00X-search-notes-by-tag/spec.md` を**下書き**します。中身を開いて確認します。
4. 続けて設計の下書きも作らせます。

   ```text
   /speckit.plan
   ```

   `plan.md` が作られ、**Constitution Check**（憲章との整合チェック）が走ります。
   設計が重要（Class B 相当）なら、AI は ADR の起票も提案します。

> **これが成功体験です。** AI が、いきなりコードを書くのではなく、
> **「何を・なぜ」（spec）→「どう作るか」（plan）** の順で、ルールに沿った下書きを出しました。

### トラック B ｜ まだ AI エージェントがない

AI なしでも、テンプレートの「品質ゲート」を体験できます。

1. 任意ツールを導入して高速チェックを走らせます（[前提環境](prerequisites.md) の「任意」）。

   ```bash
   task setup
   task verify:fast
   ```

2. `glossary.md`（用語集）に 1 行、自分のドメイン用語を追記します（これは軽微な **Class D** 変更）。
3. もう一度 `task verify:fast` を実行し、**緑（成功）** になることを確認します。

> これで「ルールは“書く”だけでなく、機械が**通らないと止める**」という仕組みを体験できました。

---

## 0:25–0:30 ｜ 何が起きたかを理解する

- できた成果物（`spec.md` など）は **AI による下書き** です。**そのままマージしません。**
- 重要変更は **人間のレビューと承認** を経て、初めて保護ブランチに入ります（[変更クラス](../concepts/governance.md)）。
- つまりあなたは今、**「AI が下書きし、人間が承認する」開発フローの 1 周目** を体験しました。

```mermaid

    you["あなたの一言<br/>（やりたいこと）"] --> ai["AI が spec を下書き"]
    ai --> review["あなたがレビュー"]
    review --> next["plan / ADR / 実装へ"]

    class review h;
```
